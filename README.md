# Интеллектуальный ассистент для поиска по базе документов

Система на базе LLM (Large Language Model) для семантического поиска и анализа корпоративных документов (договоры, приказы, служебные записки, акты, накладные). Использует RAG-архитектуру (Retrieval-Augmented Generation) с векторной базой данных ChromaDB и локальным LLM-сервером через OpenAI-совместимый API.

## Возможности

- Автоматическое извлечение метаданных из документов (номер, дата, организация, сумма, тип)
- Семантический поиск по содержимому документов
- Фильтрация по метаданным для ускорения поиска
- Кэширование ответов для повторных запросов
- Обработка опечаток в номерах договоров (латиница/кириллица, лишние пробелы)
- Объединение информации из нескольких связанных документов (договор + приказ)
- Поддержка форматов: `.md`, `.txt`, `.pdf`, `.docx`, `.doc`, `.rtf`

## Структура проекта
```
llm_glob/
├── config.py                                 # Глобальные настройки проекта, включая URL API, пути и параметры по умолчанию.
├── src/
│   ├── parse2md.py                           # Гибридный парсер PDF, извлекающий текст или использующий Vision OCR с последующей очисткой через LLM.
│   ├── cache_manager.py                      # Управляет дисковым кэшем ответов LLM для ускорения повторных запросов.
│   ├── index_manager.py                      # Создает и обслуживает векторный индекс ChromaDB для поиска по документам.
│   ├── query_processor.py                    # Обрабатывает вопросы пользователей, извлекает контекст и формирует ответы через LLM.
│   ├── prompts.py                            # Хранит шаблоны системных и пользовательских промптов для всех задач LLM.
│   ├── llm_setup.py                          # Инициализирует OpenAI-совместимый клиент с настройкой таймаутов и повторных попыток.
│   ├── metadata_extractor.py                 # Извлекает метаданные документов (автор, дата, теги) для фильтрации и индексации.
│   └── embeddings.py                         # Загружает модель эмбеддингов и генерирует векторные представления текста.
├── cleaned_md/                               # Папка для хранения очищенных Markdown-файлов после обработки PDF.
├── chroma_db/                                # Директория с файлами векторной базы данных ChromaDB и её индексами.
├── cache/                                    # Временное хранилище для кэшированных ответов и промежуточных данных.
└── test/                                     # Каталог со скриптами проверки и тестовыми данными для отладки.
```

## Требования

- Python 3.10+
- Доступ к LLM-серверу с OpenAI-совместимым API (например, vLLM, Ollama, Text Generation Inference)
- Модели:
  - Chat-модель: `openai/gpt-oss-120b` (или любая совместимая)
  - Embedding-модель: `text-embedding-nomic-embed-text-v1.5`
  - Модель с отсутствием мышления и возможностью загрузки pdf (например `qwen3:4b-instruct`)

## Конфигурация
Все настройки находятся в config.py:
```python
API_BASE_URL = "http://your-api-endpoint:port/v1"  # URL LLM-сервера
PERSIST_DIR = "./chroma_db"                         # Путь к хранилищу индекса
CACHE_DIR = "./cache"                               # Путь к кэшу ответов
MD_DIR = "./md"                                     # Путь к папке с документами
USE_CACHE = True                                    # Использовать кэш (True/False)
```

## Установка и запуск
0. Если есть файлы .pdf, то перевести их с помощью parse.py в .md

```bash
# Убедитесь, что config и parse в одной дирректории
python parse.py -i INPUT_FOLDER -o OUTPUT_FOLDER -m MODEL_NAME
```
  
1. Поместить все md в {your_dir}
2. Указать корректный API_BASE_URL в config.py
3. запустить скрипт

```bash
# С включенным кэшем (по умолчанию)
python main.py

# Без кэша — все запросы отправляются на сервер
python main.py --no-cache
```

При первом запуске автоматически:
- Создается векторный индекс в chroma_db/
- Извлекаются метаданные из всех документов
- Индекс сохраняется для повторного использования
